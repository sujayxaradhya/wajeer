# Fix Sidebar Footer Avatar and Visibility Issues

## Investigation Summary

### Issues Found

1. **Text container not hidden in collapsed state** (nav-user.tsx:78-83)
   - The `div` containing user name and email has no collapsed-state styling
   - Should be hidden when sidebar collapses to icon mode
   - Current: `<div className="grid flex-1 text-left text-sm leading-tight">`
   - Missing: `group-data-[collapsible=icon]:hidden`

2. **Avatar overflow in collapsed state** (nav-user.tsx:74 + sidebar.tsx:489)
   - Size="lg" button has `group-data-[collapsible=icon]:p-0!` (0 padding)
   - Avatar is `size-8` with no collapsed-state adjustment
   - In collapsed state: 8x8 button + 0 padding + 8x8 avatar = overflow
   - Avatar should use `group-data-[collapsible=icon]:size-8` to match button

3. **Ellipsis icon not hidden in collapsed state** (nav-user.tsx:84)
   - `EllipsisVerticalIcon` has `ml-auto size-4` but no hidden class
   - Should hide when sidebar collapsed

4. **AvatarFallback border styling inconsistency** (avatar.tsx:19)
   - Avatar has `after:border after:border-border` overlay
   - AvatarFallback overrides `rounded-lg` but base Avatar uses `rounded-full`
   - In nav-user, Avatar has `rounded-lg` but this overrides the primitive's default

### Root Cause

The nav-user component doesn't follow the shadcn sidebar pattern for collapsed state handling. The component was likely copied from a non-sidebar context without adapting for the `group-data-[collapsible=icon]` states.

## Fix Plan

### File: `apps/web/src/components/nav-user.tsx`

**Changes needed:**

1. Add collapsed-state hiding to text container:

```tsx
// Line 78: Add hidden class for collapsed state
<div className="grid flex-1 text-left text-sm leading-tight group-data-[collapsible=icon]:hidden">
```

2. Fix Avatar sizing for collapsed state:

```tsx
// Line 74: Add explicit size that works in both states
<Avatar className="size-8 rounded-lg group-data-[collapsible=icon]:size-8 group-data-[collapsible=icon]:rounded-lg">
```

Actually, the avatar is already size-8. The issue is the button padding. Let me reconsider...

The button with `size="lg"` has:

- Normal: `h-12` (12 units height)
- Collapsed: `size-8! p-0!` (forced 8x8, 0 padding)

So in collapsed state, the 8x8 avatar fills the entire button. This is actually intentional for icon-only display. The real issue is the text content not being hidden.

**Corrected fix:**

1. Hide text container in collapsed state (primary fix)
2. Hide ellipsis icon in collapsed state
3. Optionally: adjust avatar margin/padding for better visual in collapsed state

### Recommended Implementation

```tsx
// nav-user.tsx lines 74-84 - Updated
<Avatar className="size-8 rounded-lg">
  <AvatarImage src={user.avatar} alt={user.name} />
  <AvatarFallback className="rounded-lg">{initials}</AvatarFallback>
</Avatar>
<div className="grid flex-1 text-left text-sm leading-tight group-data-[collapsible=icon]:hidden">
  <span className="truncate font-medium">{user.name}</span>
  <span className="truncate text-xs text-muted-foreground">
    {user.email}
  </span>
</div>
<EllipsisVerticalIcon className="ml-auto size-4 group-data-[collapsible=icon]:hidden" />
```

Also fix the email text color - using `text-muted-foreground` instead of `text-foreground/70` for consistency.
